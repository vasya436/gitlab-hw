Задание 1. Кеширование

Приведите примеры проблем, которые может решить кеширование.

Приведите ответ в свободной форме.

Решение:

1. Медленная загрузка веб‑страниц и приложений

    Проблема: при каждом запросе сервер заново генерирует страницу (обрабатывает скрипты, обращается к БД, собирает HTML), что занимает время.

    Решение: кеширование готовых HTML‑страниц или их частей. При повторном запросе сервер отдаёт уже собранную версию из кеша.
   
2.  Высокая нагрузка на сервер и базу данных

    Проблема: повторяющиеся запросы к БД (например, на получение списка товаров или комментариев) нагружают сервер, особенно при большом трафике.

    Решение: сохранение результатов частых SQL‑запросов в кеш (например, в Redis или Memcached). Сервер берёт данные из быстрой памяти, а не выполняет запрос заново.

3. Долгие вычисления и обработка данных

    Проблема: сложные операции (агрегация статистики, построение отчётов, машинное обучение) требуют много ресурсов и времени.

    Решение: кеширование результатов вычислений. Повторный запрос отдаёт готовый результат мгновенно.

4. Ограничения внешних API

    Проблема: многие API имеют лимиты на число запросов в сутки/час или тарифицируются за каждый вызов.

    Решение: кеширование ответов API на определённое время.

5. Задержки из‑за географического расстояния

    Проблема: пользователи из удалённых регионов испытывают задержки из‑за долгого сетевого пути до основного сервера.

    Решение: CDN (Content Delivery Network) — кеширование статического контента (изображения, стили, скрипты) на серверах по всему миру.

6. Избыточный трафик между сервисами

    Проблема: в микросервисной архитектуре сервисы часто обмениваются данными, что создаёт сетевой трафик и задержки.

    Решение: кеширование промежуточных результатов в общей памяти (например, Redis).

7. Высокие затраты на хостинг и инфраструктуру

    Проблема: рост трафика требует увеличения мощности серверов, что увеличивает расходы.

    Решение: кеширование снижает нагрузку на серверы и БД, позволяя обойтись меньшими ресурсами.

8. Плохой пользовательский опыт

    Проблема: медленные страницы приводят к росту отказов (пользователь уходит, не дождавшись загрузки) и снижению конверсии.

    Решение: браузерное кеширование — стили, скрипты и изображения сохраняются локально, ускоряя повторные визиты.

9. Падение производительности при пиковых нагрузках

    Проблема: во время всплесков трафика (акции, новости, рекламные кампании) сервер может не справиться и «упасть».

    Решение: reverse proxy (например, Nginx) с кешированием — отдаёт готовые ответы из кеша, разгружая основной сервер.

10. Низкие SEO‑показатели

    Проблема: поисковые системы (особенно Google) учитывают скорость загрузки сайта при ранжировании. Медленные сайты получают более низкие позиции.

    Решение: комплексное кеширование (браузерное, серверное, CDN) ускоряет загрузку страниц.
