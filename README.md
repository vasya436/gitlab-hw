### Задание 1

На лекции рассматривались режимы репликации master-slave, master-master, опишите их различия.

Ответить в свободной форме.

Master-Slave
Принцип работы:

    Master - основной сервер, который принимает все операции записи (INSERT, UPDATE, DELETE).
    Slaves (подчиненные серверы) только читают данные и получают обновления от Master.
    Клиенты могут читать данные как с Master, так и с Slaves, что позволяет распределять нагрузку на чтение.

Master-Master
Принцип работы:

    В системе есть несколько главных серверов (Master), и каждый из них может одновременно принимать и обрабатывать запросы на запись и чтение.
    Данные между серверами автоматически синхронизируются, чтобы оставаться одинаковыми.

    Master‑Slave - оптимальный выбор для систем с преобладанием операций чтения, где важна простота настройки и возможность разгрузки основного сервера. Подходит для большинства веб‑приложений и аналитических задач.

Master‑Master - подходит для критически важных систем, требующих максимальной отказоустойчивости и балансировки нагрузки на запись. Требует более сложной настройки и решения проблем с конфликтами данных.
